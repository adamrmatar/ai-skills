# Claude Code Custom Instructions - Implementing A Simple And Effective Productivity System
> This skill teaches you how to design and implement a productivity system that balances simplicity with effectiveness, ensuring it adapts to changing needs without becoming overly complex or rigid.

## Overview

This skill focuses on creating a productivity system that is both simple and effective, avoiding the pitfalls of over-complexity and rigidity. The core idea is to establish a foundational structure that supports both shallow and deep work, allowing for flexibility and adaptability. Learn more about the core concepts in [Core Concepts](references/core_concepts.md).

## Step-by-Step Workflow

1. **Define Core Structure**: Start by creating a basic structure that is wide enough to accommodate various tasks but specific enough to be useful. Avoid over-complicating with unnecessary tags or categories.

2. **Link Workflows to Tools**: Identify the main workflows you deal with daily and connect them to specific tools. For example, use ClickUp for task management and Obsidian for personal knowledge management. Detailed guidance can be found in [Practical Guide](references/practical_guide.md).

3. **Use Links for Context**: Instead of rigid folder structures, use links to connect tasks to relevant resources. This ensures that changes in one place don’t break the entire system. See [Code Examples](references/code_examples.md) for implementation details.

4. **Prioritize Tasks**: Implement a prioritization system that helps you focus on what matters most. Avoid using low/medium/high priorities; instead, rely on intuitive understanding of task importance.

5. **Regularly Review and Simplify**: Periodically review your system to remove any unnecessary complexities. Ensure it remains streamlined and easy to use.

## Best Practices and Common Pitfalls

- **Best Practice**: Keep your system simple and flexible. Avoid adding features or tags that you don’t regularly use.
- **Pitfall**: Over-complicating your system with too many interconnected elements can lead to friction and inefficiency. For example, using too many custom fields in ClickUp can make it cumbersome to use.

## Validation and Testing Steps

1. **Test Task Retrieval**: Ensure you can quickly retrieve tasks and related resources using links.
2. **Evaluate Workflow Efficiency**: Assess whether your workflows are streamlined and free of unnecessary steps.
3. **User Feedback**: Gather feedback from team members or peers to identify any pain points or areas for improvement.

## Core Concepts

## Core Concepts

### Foundational Structure

The core of any effective productivity system is a foundational structure that is both wide enough to accommodate various tasks and specific enough to be useful. This structure should be flexible, allowing it to adapt to changing needs without becoming overly complex or rigid.

### Shallow vs. Deep Work

Shallow work refers to tasks that are necessary but not cognitively demanding, such as answering emails or updating spreadsheets. Deep work, on the other hand, involves tasks that require significant cognitive effort, such as writing a report or developing a new strategy. A good productivity system should support both types of work.

### Tool Agnostic Approach

It's important to approach productivity tools in a tool-agnostic way. Understand the workflows and concepts behind productivity first, and then choose tools that best fit those workflows. This ensures that your system remains flexible and adaptable.

### Links Over Folders

Instead of relying on rigid folder structures, use links to connect tasks to relevant resources. This ensures that changes in one place don’t break the entire system. For example, link a task in ClickUp to a Dropbox folder containing relevant files.

### Prioritization

Implement a prioritization system that helps you focus on what matters most. Avoid using low/medium/high priorities; instead, rely on intuitive understanding of task importance. This ensures that you can quickly identify and focus on critical tasks.

## Practical Guide

## Practical Guide

### Defining Core Structure

Start by creating a basic structure that is wide enough to accommodate various tasks but specific enough to be useful. Avoid over-complicating with unnecessary tags or categories. For example, in ClickUp, create a few broad categories like 'Work', 'Personal', and 'Projects'.

### Linking Workflows to Tools

Identify the main workflows you deal with daily and connect them to specific tools. For example, use ClickUp for task management and Obsidian for personal knowledge management. This ensures that each tool is used for its strengths, making your system more efficient.

### Using Links for Context

Instead of rigid folder structures, use links to connect tasks to relevant resources. This ensures that changes in one place don’t break the entire system. For example, link a task in ClickUp to a Dropbox folder containing relevant files.

### Prioritizing Tasks

Implement a prioritization system that helps you focus on what matters most. Avoid using low/medium/high priorities; instead, rely on intuitive understanding of task importance. This ensures that you can quickly identify and focus on critical tasks.

### Regular Reviews

Periodically review your system to remove any unnecessary complexities. Ensure it remains streamlined and easy to use. For example, every month, review your ClickUp setup to remove any unused custom fields or tags.
