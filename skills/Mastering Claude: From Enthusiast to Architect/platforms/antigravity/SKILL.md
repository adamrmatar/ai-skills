### Overview
Claude AI is a powerful tool that evolves from a simple conversational assistant to a full-fledged automation and architecture platform. This skill will guide you through five levels of mastery, from basic usage to advanced automation, enabling you to save time, increase productivity, and build scalable systems.

### Core Concepts
Claude operates across multiple levels, each with distinct features and capabilities:
1. **Level 1: Enthusiast** - Basic conversational use.
2. **Level 2: Beginner** - Projects, memory, and integrations.
3. **Level 3: Intermediate** - Co-work, skills, and automation.
4. **Level 4: Advanced** - Parallel workflows, sub-agents, and CLI integration.
5. **Level 5: Architect** - Cloud routines, hooks, and autonomous systems.

For detailed explanations of these concepts, refer to [Core Concepts](references/core_concepts.md).

### Step-by-Step Workflow
1. **Level 1: Enthusiast**
   - Use Claude for basic tasks like email drafting or quick scripts.
   - Leverage screenshot reading to save time.

2. **Level 2: Beginner**
   - Create your first project by organizing recurring tasks.
   - Integrate tools like Slack, Google Drive, and GitHub.
   - Use memory and past chat search to maintain context.

3. **Level 3: Intermediate**
   - Open Claude Desktop and use the co-work tab for automation.
   - Define reusable skills for repetitive tasks.
   - Schedule tasks and use mobile control for remote work.

4. **Level 4: Advanced**
   - Use sub-agents for specialized tasks like testing or documentation.
   - Leverage work trees for parallel workflows.
   - Master context window management with commands like `/compact` and `/context`.

5. **Level 5: Architect**
   - Set up cloud routines for autonomous task execution.
   - Use hooks for safety and automation.
   - Build agent teams for complex, collaborative tasks.

For a detailed workflow guide, refer to [Practical Guide](references/practical_guide.md).

### Best Practices
- **Context Management**: Use `/compact` and `/context` to optimize token usage.
- **Skill Reusability**: Build skills once and reuse them across projects.
- **Trust Building**: Start with low-stakes routines before scaling to autonomous systems.

### Common Pitfalls
- **Overloading Context**: Avoid exceeding 50% of the context window capacity.
- **Manual Management**: Automate repetitive tasks to avoid becoming a bottleneck.
- **Untrusted Sources**: Be cautious when downloading skills from unverified sources.

For more on pitfalls and how to avoid them, refer to [Common Pitfalls](references/common_pitfalls.md).

### Validation and Testing
1. **Test Skills**: Run skills in a controlled environment before deploying.
2. **Monitor Routines**: Use hooks to monitor and validate autonomous tasks.
3. **Iterate**: Continuously refine skills and routines based on feedback.
