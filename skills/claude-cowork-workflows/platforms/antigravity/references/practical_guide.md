# Practical Guide: Interactive Pair-Programming

- **Pair Programming**: Ask Claude to review your code draft and suggest refactors. You act as the driver (setting goals), and Claude acts as the navigator (checking edge cases).
- **Prototyping**: Ask Claude to generate a standalone React component as an artifact, test it in-browser, and feed back lint errors.
- **Writing Specs**: Collaborate on technical specifications by co-drafting the outline and asking Claude to flesh out details.
